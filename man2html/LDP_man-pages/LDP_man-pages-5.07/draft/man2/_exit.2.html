名前

_exit, _Exit - 呼び出し元のプロセスを終了させる  

書式

#include <unistd.h>

void _exit(int status);

#include <stdlib.h>

void _Exit(int status);

glibc 向けの機能検査マクロの要件 (feature_test_macros(7) 参照):

_Exit():

_ISOC99_SOURCE || _POSIX_C_SOURCE >= 200112L
 

説明

_exit() は、それを呼んだプロセスを「直ちに」終了させる。 その際、このプロセスが所有しているディスクリプター (descriptor) で、 オープンされているものは全てクローズされる。 プロセスの子プロセスは init(1) (もしくは prctl(2) PR_SET_CHILD_SUBREAPER 操作で定義された最も近い先祖の "subreaper" プロセス) によって継承される。 このプロセスの親プロセスに対して SIGCHLD シグナルが送出される。

status & 0xFF の値は、このプロセスの終了状態としてその親プロセスに対して返され、 wait(2) 系関数を利用することによって取得することができる。

関数 _Exit() は _exit() と等価である。  

返り値

これらの関数は値を返さない。  

準拠

POSIX.1-2001, POSIX.1-2008, SVr4, 4.3BSD. 関数 _Exit() は C99 で導入された。  

注意

exit の効果、終了状態の受渡し、ゾンビプロセス、シグナル送出などに ついての議論は、 exit(3) を参照すること。

関数 _exit() は exit(3) に似ているが、ANSI C の atexit(3) や on_exit(3) によって登録されたいかなる関数も呼び出さない。 オープンされた stdio(3) ストリームはフラッシュされない。 一方で、 _exit() はオープンされているファイルディスクリプターをクローズしないため、 未決定になっている出力がフラッシュされるのを待つのに不確定な遅れが発生する。 この遅れを発生させたくなければ、 _exit() の前に tcflush(3) のような関数を呼び出せばよい。 _exit() が呼び出されたときに、 未決定になっている全ての I/O がキャンセルされるのか、 またどの I/O がキャンセルされるのかは実装に依存する。  

C ライブラリとカーネルの違い

バージョン 2.3 より前の glibc では、 _exit() のラッパー関数は同じ名前のカーネルシステムコールを起動していた。 glibc 2.3 以降では、プロセス内の全てのスレッドを終了するために、 ラッパー関数は exit_group(2) を起動する。 (元々の _exit() システムコールは呼び出しスレッドのみを終了する。)
