名前

whereis - コマンドのバイナリ・ソース・マニュアルファイルの場所を示す  

書式

whereis [options] [-BMS directory... -f] name...  

説明

whereis は指定したファイルのバイナリ・ソース・マニュアルファイルの場所を示す。 与えられた名前は、最初に始めのパス名の部分が取り除かれ、次に (.c のような) .ext という形式の拡張子が取り除かれる。 ソースコード管理に使われている s. というプレフィックスがついたファイルも表示される。 whereis は、標準的な Linux ファイル階層中と、 $PATH と $MANPATH で指定された場所で、対象のプログラムを探す。

コマンドラインに指定された検索の制限 (オプション -b, -m, -s) は、 足し合わされて、その後に続く name パターンに適用される。 新たに検索の制限を指定すると、検索のマスクがリセットされる。 例えば、

whereis -bm ls tr -m gcc

を実行すると、"ls" と "tr" のバイナリとマニュアルページ、 "gcc" のマニュアルのみを検索する。

オプション -B, -M and -S は、 その後に続く name パターンの検索パスをリセットする。 例えば、

whereis -m ls -M /usr/share/man/man1 -f cal

を実行すると、"ls" の man ページはすべてのデフォルトパスで検索されるが、 "cal" の man ページは /usr/share/man/man1 ディレクトリのみで検索される。

 

オプション

-b
バイナリを探す。
-m
man ページを探す。
-s
ソースを探す。
-u
通常とは異なるエントリのコマンドのみを表示する。 要求されるタイプの 1 つのエントリがコマンドにない場合、 通常とは異なるコマンドと呼ばれる。 よって、 `whereis  -m  -u  *' は、カレントディレクトリのファイルからドキュメントがないもの、 または 1 つ以上のものを探す。
-B list
whereis がバイナリを探すディレクトリを制限する。 ディレクトリは空白区切りのリストで指定する。
-M list
whereis が man ページと Info 形式のドキュメントを探すディレクトリを制限する。 ディレクトリは空白区切りのリストで指定する。
-S list
whereis がソースを探すディレクトリを制限する。 ディレクトリは空白区切りのリストで指定する。
-f
直前のディレクトリリストを終了させ、 ファイル名指定の開始であることを示す。 -B, -M, -S オプションのいずれかを使う場合は、 このオプションを 使わなければならない。
-l
whereis が使う有効な検索パスを表示する。 -B, -M, -S のいずれも指定されない場合、このオプションは、 コマンドがシステム上で検索するハードコードされたパスを表示する。
-h, --help
ヘルプを表示して、終了する。
-V, --version
バージョン情報を表示して、終了する。
 

例

/usr/:bin にあるファイルから /usr/:man/:man1 に man ページがない、または /usr/:src にソースがないようなもの探す。
cd /usr/bin
whereis -u -ms -M /usr/man/man1 -S /usr/src -f *
 

ファイル検索パス

デフォルトでは whereis はハードコードされたパスでファイルを検索しようとする。 このパスは glob パターンで定義される。 このコマンドは、 $PATH と $MANPATH 環境変数をデフォルトの検索パスとして使おうとする。 どのパスが使われるかを簡単に知りたい場合、リストのオプション -l を指定すればよい。 -B, -M, -S の影響を知りたい場合も、 -l で表示できる。

 

環境変数

WHEREIS_DEBUG=all
デバッグ出力を有効にする。
 

入手方法

whereis コマンドは、util-linux パッケージの一部であり、 Linux Kernel Archive から入手できる。
